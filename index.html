Output Format

Tensor bundles with canonical JSON metadata.

TensorSuite stores each tensor as a bundle first. The *_metadata.json file is the canonical schema for tensor attributes and metadata; .tns and .tnsb are data representations inside the bundle. The JSON file stays comment-free; attribute meanings and allowed values are documented below.

Bundle Layout

One folder per tensor record.

A bundle includes a tensor-name metadata file, data files, a README, and any additional files the dataset needs.

Named Files

A tensor-name folder uses matching file prefixes, such as nell-2_metadata.json, nell-2.tns, and nell-2.tnsb.

Binary Tensor

.tnsb stores tensor entries in binary form for faster loading. Large bundles may provide only .tnsb as the tensor data file, with files.text set to null.

Auxiliary Data

Auxiliary data can live in the metadata JSON or as additional files in the bundle, documented by README.md.

TNSB Header

.tnsb keeps the same line header pattern as .tns; the binary payload begins immediately after the size-line newline, so header_length is not required for sequential loading.

.tns File Format

Optional readable coordinate data.

The text file keeps a minimal cross-check header, one size line, and coordinate entries. Full attributes and metadata stay in the bundle metadata JSON.

Minimal Header

The first line is %%TensorSuite-TNS. Then % version and % name let readers cross-check the file against the metadata JSON.

Metadata JSON

group, id, time, source, size, type, ordering, and file mappings are defined in *_metadata.json.

Size Line

One line gives the tensor order, all mode dimensions, and NNZ so readers can preallocate memory and validate the entry count.

Entries

Each data row stores one-based or zero-based coordinates followed by a value. 

%%TensorSuite-TNS
% version: 1.0
% name: nell-2
3 12092 9184 28818 76879419
1 1 1 1.0000000000000000
1 1 2 2.0000000000000000
%%TensorSuite-TNSB
% version: 1.0
% name: nell-2
3 12092 9184 28818 76879419
[binary coordinate/value records begin here]

Attribute Values

Allowed values for tensor attributes.

Metadata fields and tensor size fields are not listed here. They are free-form identity fields or direct size information.

Attribute Allowed Values Meaning
storage coordinate Coordinate entries: indices followed by one value.
value_type float32, float64, int32, int64, complex64, complex128 Numeric type used by tensor values.
index_type uint32, uint64 Integer type used by binary coordinate indices.
endianness little, big Byte order used by binary numeric records.
index_base 0, 1 Whether coordinates are zero-based or one-based.
sorted none, lexicographic Whether entries are sorted.
sorted_order Permutation of mode ids, such as [1,2,3] Mode priority used for lexicographic sorting.
duplicates allowed, disallowed, summed Policy for repeated coordinates.
explicit_zeros allowed, disallowed Whether stored entries may have numerical value zero.
pattern_symmetry full, partial, no Symmetry of the nonzero coordinate pattern.
numerical_symmetry full, partial, no Symmetry of both coordinate pattern and mirrored values.
sparsity_type no, semi, element, block Dense, semi-sparse, element-sparse, or block-sparse structure.
dense_modes Mode-id list, such as [] or [1,2] Modes that are dense or nearly dense.

Synthetic Tensor Generation

Controlled sparse tensor generators for experiments.

TensorSuite writes .tns coordinate-format tensors whose non-zero values are sampled from a standard normal distribution. The limited generator estimates memory use before writing data.
